Case Study — Operations · Forecasting · Seattle

Seattle Event-Driven
Demand Forecasting

Predicting traffic volume surges, transit ridership spikes, and parking demand around Seattle’s major event corridors — using public data, time-series modeling, and directional flow analysis.

Type Portfolio Project
Domain Operations · Transportation · Forecasting
Geography King County, WA
Stack Python · GTFS · WSDOT API · Folium

Problem

Seattle hosts 200+ major events annually across Climate Pledge Arena, T-Mobile Park, Lumen Field, and KeyArena — each generating predictable but unmanaged demand shocks on surrounding road corridors and transit lines. Operations teams at logistics companies, transit agencies, and venue operators lack a unified model for anticipating these surges 24–72 hours in advance.

The question this project answers: given a confirmed event schedule, what does demand look like corridor-by-corridor in the 2–4 hours before and after the event? And which King County road segments cross critical utilization thresholds?

Data Sources

WSDOT Traffic Counts Hourly vehicle counts at 500+ sensor stations across King County road network
King County Metro GTFS Real-time and scheduled transit feeds — stops, routes, frequency by hour
Seattle Special Events Office Public permitted event calendar — venue, date, expected attendance, road closures
Historical game/concert data Seahawks, Mariners, Kraken, major concerts — scraped attendance and start times
OpenStreetMap / OSMnx Road network topology for directional flow modeling
NOAA weather data Precipitation and temperature — strong predictor of transit vs. vehicle modal split

Methodology

The model treats each event as a demand perturbation layered on top of baseline traffic. The pipeline runs in three stages:

  1. Baseline extraction. WSDOT hourly counts for each sensor station, decomposed by day-of-week and time-of-day to establish a rolling 90-day baseline. Seasonality and trend removed via STL decomposition.
  2. Event feature engineering. Each confirmed event is encoded as: venue, expected attendance bucket (5K / 10K / 20K+), day type, start time, and weather forecast. Proximity-weighted influence radius assigned per venue (500m–2km buffer).
  3. Demand lift forecasting. Gradient boosting regressor (XGBoost) trained on 3 years of event+traffic pairs to predict percent volume lift per sensor station per hour. Outputs a demand lift surface across the road network for any upcoming event.
  4. Transit impact overlay. GTFS scheduled frequency compared against predicted ridership uplift to flag under-served corridors — routes where predicted demand exceeds scheduled capacity by >15%.
  5. Directional flow modeling. OSMnx network graph used to model directional ingress/egress flow from event venues — identifies which road segments will carry the majority of the surge load.

The interactive Folium map visualizes predicted demand lift by road segment for any given event date — shading corridors by forecast utilization level (green / amber / red). Transit under-service flags appear as highlighted routes. A summary table outputs the top 10 road segments projected to exceed capacity thresholds.

Business Impact

This model answers operational questions relevant to multiple Seattle stakeholders:

  • Logistics & last-mile operators — avoid event corridors or pre-position vehicles; re-route last-mile delivery to avoid peak congestion windows
  • Transit agencies (King County Metro) — proactively add service frequency on flagged under-served routes before events
  • Venue operations teams — time parking garage open/close windows and staff arrivals against predicted ingress surge
  • Rideshare/TNC operators — driver positioning recommendations by predicted demand zone
